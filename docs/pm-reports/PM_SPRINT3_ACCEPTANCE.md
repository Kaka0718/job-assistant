## 4. 核心功能验证

### 4.1 岗位管理流程

```
用户 -> 岗位列表 -> 新建/编辑 -> 表单(名称/方向/技能标签/标签/备注/分析/面试问题)
    -> 保存(校验) -> 列表更新 -> 搜索/筛选/归档/删除(ConfirmDialog)
```

### 4.2 投递看板流程

```
用户 -> 投递列表 -> 列表/看板切换
    +-- 列表: 搜索 + 状态筛选 + 分页(20条/次)
    +-- 看板: 8 列 Kanban(草稿->已投递->已读->沟通中->面试中->Offer->拒绝->已归档)
        -> 内联状态流转 -> 乐观更新 -> 成功/回滚 -> 详情页
```

### 4.3 仪表盘流程

```
页面加载 -> 统计卡片(今日/本周/平均匹配度/有进展) + 最近 5 条投递
    -> 首次使用引导(完善档案 -> 配置 API Key -> 生成打招呼)
```

### 4.4 错误处理覆盖

| 场景 | 处理方式 |
|:---|:---|
| 加载中 | Skeleton 骨架屏（所有页面） |
| 空数据 | EmptyState + 引导按钮 |
| 删除操作 | ConfirmDialog 弹窗确认 |
| 保存失败 | Toast 错误提示 |
| 状态更新失败 | 乐观更新回滚 + Toast 提示 |
| 编辑模式加载失败 | Toast 提示 + 跳转列表 |
| 仪表盘加载失败 | 错误卡片 + 重试按钮 |
| 首次使用 | 欢迎引导卡片 + 操作按钮 |
