## 2. 模块拆分与开发指令

### 模块 D1：投递看板拖拽排序

- **前端 (Frontend) 任务清单**：
    1. **安装依赖**：`npm install @dnd-kit/core @dnd-kit/sortable @dnd-kit/utilities`
    2. **看板列拖拽**：
       - 引入 `@dnd-kit/core` 的 `DndContext`、`DragOverlay`
       - 卡片在列内可拖拽排序（垂直拖动）
       - 跨列拖拽（可选）：将卡片从一列拖到另一列，自动触发状态更新
       - 拖拽时显示半透明 DragOverlay
    3. **排序持久化**：
       - 列内排序：前端本地排序，不涉及后端（投递记录无顺序概念）
       - 跨列拖拽：调用 `updateApplicationStatus(id, newStatus)` 持久化状态变更
    4. **边界逻辑**：
       - 拖拽中卡片显示抬起阴影 + 缩放效果
       - 拖拽到非法区域回弹
       - 拖拽状态更新失败时回滚到原始位置

- **后端 (Backend) 任务清单**：
    1. 本模块无后端任务（状态更新接口已存在）

### 模块 D2：仪表盘数据可视化

- **前端 (Frontend) 任务清单**：
    1. **安装依赖**：`npm install recharts`
    2. **匹配度分布柱状图**：
       - 在 Dashboard 新增"匹配度分布"卡片
       - 横轴：分数段（0-20 / 21-40 / 41-60 / 61-80 / 81-100）
       - 纵轴：投递数量
       - 使用 `@radix-ui/react-tabs` 或 Select 切换图表类型（柱状图/折线图）
    3. **每周投递趋势折线图**：
       - 横轴：最近 8 周
       - 纵轴：每周投递数量
       - 展示趋势变化
    4. **状态分布环形图**：
       - 展示各状态的投递数量占比
       - 使用 Recharts 的 `PieChart` + `Cell`，颜色与设计系统状态色对齐
    5. **边界逻辑**：
       - 无数据时显示 EmptyState 而非空图表
       - 图表加载中显示 Skeleton 占位
       - 图表响应式：窗口缩小时自适应

- **后端 (Backend) 任务清单**：
    1. **新增趋势数据接口**（可选）：
       - `commands/dashboard.rs` 新增 `get_weekly_trends()` 命令
       - 返回 `Vec<WeeklyTrend>`：`{ week_start: String, count: u32 }`
       - 或前端直接从 `list_applications()` 聚合计算（推荐，性能足够）

### 模块 D3：岗位档案批量导入/导出

- **前端 (Frontend) 任务清单**：
    1. **导出功能**：
       - 在岗位列表页（PositionListPage）增加"导出"按钮
       - 导出格式：JSON 或 CSV
       - 调用后端导出接口，或前端直接序列化当前列表数据
       - 导出后触发浏览器下载（Tauri 中保存到用户选择的路径）
    2. **导入功能**：
       - 在岗位列表页增加"导入"按钮
       - 点击后弹出文件选择器（`.json` / `.csv`）
       - 解析文件内容 → 批量调用 `createPosition(data)`
       - 导入进度提示（当前第 N 条 / 共 M 条）
    3. **边界逻辑**：
       - 导入文件格式校验：非 JSON/CSV 文件提示错误
       - 导入数据字段校验：缺少必填字段时跳过 + 记录错误日志
       - 导入中的重复检测：标题相同是否覆盖/跳过（默认跳过）
       - 导入完成后 Toast 汇总（成功 N 条，跳过 M 条）

- **后端 (Backend) 任务清单**：
    1. 本模块无后端任务（前端直接序列化/反序列化，或通过 Tauri dialog 保存文件）

### 模块 D4：数据备份/恢复

- **前端 (Frontend) 任务清单**：
    1. **备份功能**：
       - 在设置页（SettingsPage）增加"数据管理"区域
       - "导出备份"按钮：将整个 `data/` 目录打包为 `.zip` 文件
       - 保存到用户选择的路径
    2. **恢复功能**：
       - "导入备份"按钮：选择 `.zip` 文件
       - 解压后覆盖 `data/` 目录
       - 恢复前弹出 ConfirmDialog 警告"将覆盖现有数据"
    3. **边界逻辑**：
       - 备份中显示进度条
       - 恢复前显示备份文件信息（日期、数据量）
       - 恢复失败时保留原数据不变
       - 备份文件格式校验：非 zip 或格式错误提示

- **后端 (Backend) 任务清单**：
    1. **备份命令**（`commands/backup.rs`）：
       - `export_backup(path: String)` → 将 `data/` 目录打包为 zip 到指定路径
       - 使用 Rust 的 `zip` crate
    2. **恢复命令**：
       - `import_backup(path: String)` → 解压 zip 覆盖 `data/` 目录
       - 解压前校验 zip 文件结构
       - 先备份当前数据到临时目录，恢复失败时回滚
    3. **Cargo.toml 新增依赖**：
       - `zip = "2"` — zip 打包/解压
    4. **非功能性要求**：
       - 备份/恢复操作在异步线程执行，不阻塞 UI
       - 大文件备份时显示进度回调

## 4. 二次验证关注点

1. **拖拽体验**：拖拽跟手度、跨列动画流畅度、回弹动画
2. **图表数据准确性**：图表数据与仪表盘统计卡片数据是否一致
3. **导入数据完整性**：导入后所有字段是否完整保留，中文字符是否乱码
4. **备份恢复安全性**：恢复失败时原数据是否完好无损
5. **大文件处理**：大量岗位档案（100+）导入/导出是否超时或卡死

## 5. 开发顺序建议

```
D1 (拖拽排序) ──→ 纯前端，dnd-kit 集成
D2 (数据可视化) ──→ 纯前端，Recharts 集成
D3 (导入导出) ──→ 前端为主，少量后端
D4 (备份恢复) ──→ 前后端配合，后端 zip 打包

建议并行：D1 + D2 可同时开发（互不依赖）
           D3 + D4 可同时开发（互不依赖）
```
