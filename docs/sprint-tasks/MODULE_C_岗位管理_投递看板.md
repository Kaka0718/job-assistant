### 模块 C2：投递记录看板视图 + 状态流转

- **前端 (Frontend) 任务清单**：
    1. **ApplicationListPage.tsx — 看板视图**：
       - 支持两种视图切换：列表视图 / 看板（Kanban）视图
       - 列表视图：当前实现（卡片列表 + 搜索），增加分页（每页 20 条）
       - 看板视图：按状态分组（草稿/已投递/已读/沟通中/面试中/Offer/拒绝/已归档）
       - 看板列：每列显示对应状态的卡片，可上下拖动排序（可选）
       - 看板列标题：状态名 + 数量计数
    2. **状态流转交互**：
       - 点击卡片上的状态 Badge，弹出状态选择器（Select 下拉）
       - 选择新状态后调用 `updateApplicationStatus(id, status)`
       - 状态变更后 Toast 提示"状态已更新"
       - 看板模式下，状态变更后卡片自动移动到对应列
    3. **投递详情页**（可选，如有时）：`/applications/:id`
       - 展示完整投递信息：公司、岗位、JD 原文、打招呼文案、匹配度分析、状态流转历史
       - 可编辑状态和 hasProgress 标记
    4. **筛选功能**：
       - 状态筛选：多选下拉
       - 日期范围筛选：日期选择器
       - 搜索：公司名/岗位名模糊搜索
    5. **边界逻辑**：
       - 加载中：Skeleton 列表
       - 空数据：EmptyState + "去生成打招呼"引导
       - 状态更新失败：Toast 错误提示 + 回滚 UI
       - 分页：加载更多按钮或自动滚动加载

- **后端 (Backend) 任务清单**：
    1. 本模块无后端任务（Sprint 2 B1 已实现 `list_applications(filter)` + `update_application_status`）

### 模块 C3：仪表盘数据可视化

- **前端 (Frontend) 任务清单**：
    1. **Dashboard.tsx — 实时数据绑定**：
       - 统计卡片接入真实数据：
         - 今日投递：调用 `applicationStore.fetchApplications()` 筛选今日记录
         - 本周投递：筛选本周记录
         - 平均匹配度：计算所有记录的 matchScore 平均值
         - 有进展数量：统计 hasProgress = true 的记录数
       - 最近投递列表：展示最近 5 条投递记录（右侧或下方区域）
       - 匹配度分布：简单柱状图或进度条展示各分数段分布
    2. **状态覆盖**：
       - 加载中：统计卡片 Skeleton + 列表 Skeleton
       - 空数据：欢迎引导页（当前已有），提示"完善个人档案，开始求职之旅"
       - 有数据：统计看板 + 最近投递列表
       - 错误：错误提示卡片 + 重试按钮
    3. **边界逻辑**：
       - 首次使用：引导用户完善档案 + 配置 API Key
       - 数据刷新：页面聚焦时自动刷新（可选）
       - 无匹配度数据：显示 "--" 而非 0

- **后端 (Backend) 任务清单**：
    1. **新增统计查询接口**（可选）：
       - `commands/dashboard.rs` — 新增 `get_dashboard_stats()` 命令

## 4. 二次验证关注点

1. **CRUD 数据一致性**：新建/编辑/删除后重新读取，确保数据不丢失
2. **状态流转准确性**：投递状态更新后，看板列之间移动是否正确
3. **仪表盘数据准确性**：统计数字与投递记录列表是否一致（今日投递数 = 当天创建的记录数）
4. **删除安全**：删除前是否有 ConfirmDialog 确认，删除后是否可从文件系统恢复
5. **看板响应式**：看板列在窗口缩小时是否正常滚动，无布局断裂
